Intro

This project uses the fetch API to communicate asynchronously with Rails in JSON.

MenuGenerator App

Biggest Challenges

It’s probably not a surprise that getting the frontend and backend to exchange data was the most difficult aspect of this project.

On the backend, a major hurdle was formatting the JSON response to the GET request so each restaurant is passed as an object containing an array of its own dishes.

Actual JSON from the server

This data structure is what allows me to present it to the user on the frontend in a way that makes sense.

Displaying the data received and attaching event listeners to the buttons presented another series of challenges.

Adding event listeners to the dynamically generated buttons and attaching fetch API methods to them were probably the biggest hurdles to overcome on the frontend.

Essentially the entire UI is dynamically generated so I had to build POST and DELETE requests in to certain elements after using

The add buttons being dynamically generated with event listeners that perform POST requests

Conclusion

Deceptively simply in its appearance, this project required quite a bit of craftsmanship in order to have what are essentially completely separate front and back ends share information.
